## Installation

```npm
npx shadcn@latest add https://r.fiberui.com/r/hooks/use-interval.json
```

A declarative wrapper around `setInterval`. The interval is automatically cleaned up when the component unmounts or when the delay changes, and the callback always sees the latest state — no stale closures.

<Callout type="info" title="Source Code">
    View the full hook implementation in the [Hook Source
    Code](#hook-source-code) section below.
</Callout>

## Features

- **Stale Closure Safe** — Always executes the latest callback without restarting the timer
- **Pausable** — Pass `null` as the delay to pause the interval
- **Auto-Cleanup** — Clears the interval on unmount or when the delay changes
- **Zero Dependencies** — Uses only React's built-in hooks

## Common Patterns

### Basic Tick

```tsx
useInterval(() => {
    setCount((c) => c + 1);
}, 1000);
```

### Pausable

```tsx
// Pass null to pause the interval entirely
useInterval(() => fetchData(), isPaused ? null : 3000);
```

### Dynamic Speed

```tsx
// Change the delay to speed up or slow down
const [speed, setSpeed] = useState(1000);

useInterval(() => tick(), speed);
```

## API Reference

### Hook Signature

```tsx
function useInterval(callback: () => void, delay: number | null): void;
```

### Parameters

| Name       | Type             | Description                                     |
| ---------- | ---------------- | ----------------------------------------------- |
| `callback` | `() => void`     | The function to execute on every tick.          |
| `delay`    | `number \| null` | Interval in milliseconds. Pass `null` to pause. |

### Returns

`void` — This hook does not return anything. Control the interval by changing the `delay` value (set to `null` to pause, set to a number to start/resume).

<Callout type="warn" title="Delay of 0">
    A delay of `0` is valid — it schedules the callback for the next event-loop
    tick. Only `null` prevents the interval from being created.
</Callout>

<MdxAccordions type="single">
<MdxAccordion title="useInterval vs useTimeout">

| Feature       | `useInterval`                 | `useTimeout`                         |
| ------------- | ----------------------------- | ------------------------------------ |
| **Execution** | Repeats on every tick         | Fires once after the delay           |
| **Returns**   | `void`                        | `{ clear, reset }`                   |
| **Pause**     | Set delay to `null`           | Set delay to `null`                  |
| **Reset**     | Change delay value            | Call `reset()`                       |
| **Best For**  | Polling, counters, animations | Delayed actions, debounce, one-shots |

</MdxAccordion>
</MdxAccordions>
